The problem

Traditional security tools miss the AI layer

As organisations deploy AI tools like Microsoft 365 Copilot and Azure OpenAI-powered assistants, the attack surface shifts. The question is no longer just what the AI can do — it's what data it can access, what it might leak, and whether anyone is auditing its behaviour in production.

📄

PII flows into RAG pipelines unscreened

Patient notes, HR files, and contracts get embedded into vector databases raw. The LLM retrieves and echoes SSNs, DOBs, and account numbers to anyone querying — including people who shouldn't have access.

💬

LLM outputs echo sensitive data

Models repeat PII from their context window in responses. Without output scanning, this leakage reaches end users and gets captured in plaintext application logs.

🤖

Copilot and AI assistants lack behavioural audits

Tools like Microsoft 365 Copilot access org-wide data. Without behavioural audits, data access boundary reviews, and sensitivity label enforcement, exposure is silent and cumulative.

⚙️

IAM policies grant AI agents too much access

Azure RBAC roles with wildcard permissions on Key Vault, Storage, and SQL create excessive agency risk — especially as AI agents begin taking autonomous actions on behalf of users.

The toolkit

Four audit modules. One command.

Every module runs locally — no API calls, no cloud exposure, no ongoing cost. Each produces a structured JSON audit log alongside the terminal report for compliance record-keeping.

Module 01

PII Scanner

Detects 15+ entity types — SSN, PHI, emails, IPs, credit cards — against HIPAA Safe Harbor §164.514(b)(2). Built on Microsoft Presidio with custom recognisers tuned for AI pipeline contexts including RAG ingestion, LLM outputs, and application logs.

HIPAA · NIST MAP 5.1 · OWASP LLM06
Module 02

Prompt Guard

30 adversarial payloads across 7 OWASP LLM categories — direct injection, RAG indirect injection, jailbreaks, role boundary bypass, system prompt extraction, excessive agency, and hallucination boundary tests. Offline mode requires zero API calls.

OWASP LLM01 / LLM06 / LLM08 / LLM10
Module 03

Governance Scorecard

Scores AI policy documents and system prompts against 24 weighted checks across NIST AI RMF (GOVERN / MAP / MEASURE / MANAGE) and OWASP LLM Top 10 mitigations. Pure Python stdlib — no dependencies beyond terminal formatting.

NIST AI RMF · OWASP LLM Top 10
Module 04

IAM Validator

Audits Azure RBAC role definitions and Conditional Access policies for least-privilege violations. Detects wildcard permissions, privilege escalation paths, Key Vault and SQL exposure, missing MFA enforcement, and root-level assignable scopes.

CIS Azure · Zero-Trust · OWASP LLM07/LLM08

Vector database security — what airadar addresses

Vector databases introduce a security layer most teams haven't thought through. Embeddings are not anonymous — sensitive data encoded as vectors can be reconstructed through similarity search, and PII ingested at embedding time persists silently in the store.

→Privacy leakage via similarity search — queries can retrieve chunks containing PII even when the user has no direct document access
→Data poisoning via RAG injection — malicious content in retrieved documents can manipulate LLM responses at query time
→Pre-ingestion PII gate — Module 01 scans documents before embedding, blocking non-compliant content from entering the vector store
→Indirect injection testing — Module 02 RAG payloads test whether retrieved document content can override LLM instructions

Real-world scenarios

Where airadar fits in your pipeline

RAG pipeline audit

Before vector database ingestion

Scan documents before embedding. Block non-compliant files from entering your knowledge base. Catches PII that would otherwise persist silently in vector stores and leak through similarity search.

🔴 HIGH risk if skipped
LLM output monitoring

After model response generation

Scan AI responses before returning to users. Catches PII echoed from context window, sensitive data in completions, and content that should have been filtered by Azure AI Content Safety.

🔴 HIGH risk if skipped
System prompt audit

In your CI/CD deployment pipeline

Lint system prompts before deployment. Catches hardcoded credentials, PII, and API keys that get sent to the LLM on every single request — including test accounts left in production configs.

🔴 HIGH risk if skipped
Vendor data review

Before sharing with third-party AI vendors

HIPAA requires de-identification before PHI leaves your control. Run airadar as a final compliance gate before exporting data to external AI providers or fine-tuning services.

⚠ MEDIUM risk if skipped
